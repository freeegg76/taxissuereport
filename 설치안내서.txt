 1. 개요
================================================================================

세무 AI 어시스턴트는 판례 기반 세무 이슈 분석 보고서를 자동 생성하는 시스템입니다.

  · 세무 이슈를 자연어로 입력하면 관련 판례를 자동 검색합니다.
  · 검색된 판례를 AI가 분석하여 구조화된 보고서를 생성합니다.
  · 생성된 보고서는 피드백을 통해 수정할 수 있으며,
    Word(.docx) 또는 PDF 파일로 내보낼 수 있습니다.

※ 본 프로그램은 AI가 생성한 참고 자료를 제공하며, 법적 조언이 아닙니다.
   최종 세무 판단은 반드시 세무사 또는 변호사와 상담하시기 바랍니다.

 2. 시스템 요구사항
================================================================================

  운영체제  :  Windows 10 (버전 1709 이상) 또는 Windows 11
  메모리    :  최소 8GB RAM (16GB 권장)
  저장공간  :  최소 5GB 이상의 여유 공간
  인터넷    :  설치 중 및 사용 중 인터넷 연결 필요
  포트      :  8001번 (백엔드), 3000번 (프론트엔드) 포트가 사용 중이지 않아야 함

  ※ Python 3.9 이상 및 Node.js 18 이상이 필요하나,
     설치 프로그램이 자동으로 설치해 드립니다.

 4. 설치 방법
================================================================================

【 방법 1 】 install 폴더만 복사하여 설치 (가장 간단)
─────────────────────────────────────────────

  설치 파일 3개만 있으면 됩니다. 소스코드는 자동으로 다운로드됩니다.

  1. 아래 세 파일을 새 PC에 복사합니다.
       install\install.bat
       install\install.vbs
       install\install.ps1

     예) USB 또는 공유 폴더에서 복사
         복사 위치 예) C:\세무AI\install\install.bat
                       C:\세무AI\install\install.vbs
                       C:\세무AI\install\install.ps1

  2. [ install.bat ] 파일을 더블클릭합니다.
     install.bat 실행 시 화면이 즉시 닫히거나 오류가 발생하면
     대신 [ install.vbs ] 를 더블클릭하세요. (아래 FAQ 참조)

  3. 소스코드를 GitHub에서 자동으로 다운로드한 후 설치를 계속 진행합니다.
     (인터넷 연결 필요)


【 방법 2 】 GitHub에서 직접 다운로드
─────────────────────────────────────────────

  1. 웹 브라우저에서 아래 주소로 이동합니다.
     https://github.com/freeegg76/taxissuereport

  2. 초록색 [ Code ] 버튼 → [ Download ZIP ] 선택

  3. ZIP 파일을 원하는 위치에 압축 해제합니다.
     예) C:\taxissuereport\

  4. 압축 해제된 폴더의 [ install ] 폴더 → [ install.bat ] 더블클릭


【 방법 3 】 Git을 이용한 복제
─────────────────────────────────────────────

  Git이 설치되어 있는 경우 명령 프롬프트(cmd)에서 아래를 실행합니다.

  > git clone https://github.com/freeegg76/taxissuereport.git
  > cd taxissuereport\install
  > install.bat


【 설치 진행 과정 】
─────────────────────────────────────────────

  install.bat 실행 후 아래 7단계가 자동으로 진행됩니다.

  [1/7] 소스코드 확인
        · 소스코드가 없으면 GitHub에서 자동으로 다운로드합니다.
        · 이미 있으면 이 단계를 건너뜁니다.

  [2/7] Python 확인
        · Python 3.9 이상이 설치되어 있으면 그대로 사용합니다.
        · 설치되어 있지 않으면 Python 3.12 를 자동 설치합니다.

  [3/7] Node.js 확인
        · Node.js 18 이상이 설치되어 있으면 그대로 사용합니다.
        · 설치되어 있지 않으면 Node.js 20 LTS 를 자동 설치합니다.

  [4/7] 백엔드 설정
        · Python 가상환경을 생성합니다.
        · 필요한 Python 패키지를 설치합니다.
        · 데이터베이스를 초기화합니다.

  [5/7] 프론트엔드 설정
        · 필요한 Node.js 패키지를 설치합니다.

  [6/7] 프론트엔드 빌드
        · 화면을 프로덕션 빌드합니다. (1~3분 소요)

  [7/7] 실행 파일 생성
        · start.bat 및 stop.bat 을 프로젝트 루트 폴더에 생성합니다.

  설치 완료 후 "지금 바로 시작하시겠습니까? (Y/N)" 메시지가 나타납니다.
  Y 를 입력하면 서비스가 즉시 시작됩니다.

  ※ 최초 설치 시 전체 소요 시간 : 약 5~15분 (인터넷 속도에 따라 다름)
  ※ 설치 중 Windows 보안 경고가 나타나면 [ 허용 ] 을 클릭하세요.

 5. 첫 실행 및 계정 설정
================================================================================

  설치 완료 후 서비스가 시작되면 웹 브라우저가 자동으로 열리고
  아래 설정 화면이 나타납니다.

  주소 : http://localhost:3000/config

  ┌───────────────────────────────────────────────────────────┐
  │  Configuration                                            │
  │                                                           │
  │  LLM Provider                                             │
  │  [ Google Gemini ]  [ Anthropic Claude ]                  │
  │                                                           │
  │  Google API Key (또는 Anthropic API Key)                  │
  │  [ 발급받은 API 키 입력                              ]    │
  │                                                           │
  │  LLM 모델                                                 │
  │  [ gemini-2.5-flash                                  ]    │
  │                                                           │
  │  국가법령정보 API Key (LAW_API_KEY)                       │
  │  [ 법제처 API 키 입력                                ]    │
  │                                                           │
  │  [ 저장 및 적용 ]                                        │
  └───────────────────────────────────────────────────────────┘

  ① LLM Provider 를 선택합니다. (Google Gemini 권장)
  ② 해당 제공자의 API 키를 입력합니다.
  ③ 국가법령정보 API 키를 입력합니다.
  ④ [ 저장 및 적용 ] 버튼을 클릭합니다.
  ⑤ "서버 재시작 중..." 메시지가 나타난 후 자동으로 완료됩니다.
  ⑥ 설정 완료 후 홈 화면에서 바로 사용하실 수 있습니다.

  ※ 설정값은 backend/.env 파일에 저장됩니다.
  ※ API 키는 로컬 서버에만 저장되며 외부로 전송되지 않습니다.

 6. 일상적인 사용 방법
================================================================================

【 서비스 시작 】
─────────────────────────────────────────────

  프로젝트 루트 폴더(설치 위치)의 [ start.bat ] 을 더블클릭합니다.

  · 백엔드 서버 (포트 8001) 와 프론트엔드 서버 (포트 3000) 가 함께 시작됩니다.
  · 잠시 후 웹 브라우저가 자동으로 열립니다.
  · 이미 API 키가 설정된 경우 홈 화면으로, 미설정 시 설정 화면으로 이동합니다.

  작업표시줄에 두 개의 명령 프롬프트 창이 생깁니다.
    - "세무AI-백엔드" : 백엔드 서버 로그
    - "세무AI-프론트엔드" : 프론트엔드 서버 로그


【 서비스 종료 】
─────────────────────────────────────────────

  프로젝트 루트 폴더의 [ stop.bat ] 을 더블클릭합니다.

  · 실행 중인 모든 서비스가 즉시 종료됩니다.
  · 명령 프롬프트 창을 직접 닫아도 종료됩니다.


【 보고서 작성 흐름 】
─────────────────────────────────────────────

  ① 홈 화면에서 세무 이슈를 자연어로 입력합니다.
     예) "법인이 대표이사에게 시가보다 낮은 가격으로 부동산을 양도한 경우
          부당행위계산부인 적용 여부를 검토해 주세요."

  ② [ 분석 시작 ] 버튼을 클릭합니다.

  ③ 4단계가 자동으로 순차 실행됩니다.
     STEP 1 : 이슈 분석   — 세목, 쟁점, 검색 전략 추출
     STEP 2 : 판례 검색   — 국가법령정보 API로 관련 판례 검색
     STEP 3 : 판례 선별   — AI가 관련 판례를 선별
     STEP 4 : 보고서 생성 — AI가 분석 보고서 작성

  ④ 완료되면 보고서가 화면에 표시됩니다. (약 2~5분 소요)

  ⑤ 보고서 내용을 검토한 후 필요 시 피드백을 입력합니다.
     예) "결론 섹션을 더 명확하게 작성해 주세요."
         "대법원 판례 위주로 재구성해 주세요."

  ⑥ 최종 확인이 끝나면 [ 최종 확정 ] 버튼을 클릭합니다.

  ⑦ [ 내보내기 ] 드롭다운에서 Word(.docx) 또는 PDF 로 저장합니다.

 7. 업데이트 방법
================================================================================

새 버전이 배포된 경우, 설치 데이터(API 키, 보고서 이력)를 유지한 채로
소스코드만 최신 버전으로 교체할 수 있습니다.

【 업데이트 순서 】
─────────────────────────────────────────────

  1. 서비스가 실행 중이라면 [ stop.bat ] 으로 먼저 종료합니다.
     (update.bat 이 자동으로 종료하므로 생략해도 됩니다.)

  2. install 폴더의 [ update.bat ] 을 더블클릭합니다.

  3. 아래 5단계가 자동으로 진행됩니다.

     [1/5] 서비스 중지       — 실행 중인 서버를 자동으로 종료
     [2/5] 최신 소스 다운로드 — GitHub에서 최신 버전 다운로드
     [3/5] 소스 코드 업데이트 — 변경된 파일만 덮어쓰기
     [4/5] 백엔드 업데이트   — pip 패키지 업데이트 + DB 마이그레이션
     [5/5] 프론트엔드 업데이트— npm 패키지 업데이트 + 화면 재빌드

  4. 완료 후 "지금 바로 서비스를 시작하시겠습니까? (Y/N)" 메시지에
     Y 를 입력하면 서비스가 즉시 시작됩니다.

  ※ 업데이트 소요 시간 : 약 2~5분 (인터넷 속도에 따라 다름)

 8. 주요 기능 소개
================================================================================

  ■ 보고서 구조 (6개 섹션)

    2. 판례 분석        — 관련 판례 원문 인용 및 해석

    5. 실무 대응안      — 즉시 조치 및 중기 대응 방안
    6. 결론             — 비전문가도 이해할 수 있는 요약

  ■ 할루시네이션 방지 3단계 검증
    · 판례 직접 인용  : [대법원 2019두38656] 형식 각주 표시
    · AI 의견 구분   : "※ 검토 의견:" 블록으로 시각적 구분
    · 법령 참조      : "소득세법 제19조 제1항" 형식 (법령명 필수)

  ■ 이슈 목록 관리
    사이드바 "이슈 목록" 에서 모든 과거 분석 이력을 조회할 수 있습니다.

  ■ 내보내기 형식
    · Word (.docx) : 편집 가능한 문서, 한글 폰트(맑은 고딕) 지원
    · PDF         : 인쇄용 고정 레이아웃

 9. 자주 묻는 질문 (FAQ)
================================================================================

Q. 설치 후 브라우저가 자동으로 열리지 않습니다.
A. 웹 브라우저를 직접 열고 아래 주소를 입력하세요.
   http://localhost:3000

Q. "Python을 찾을 수 없습니다" 오류가 나타납니다.
A. Python을 수동으로 설치 후 다시 install.bat 을 실행하세요.
   다운로드 : https://www.python.org/downloads/
   설치 시 "Add Python to PATH" 체크박스를 반드시 선택하세요.

Q. "Node.js를 찾을 수 없습니다" 오류가 나타납니다.
A. Node.js를 수동으로 설치 후 다시 install.bat 을 실행하세요.
   다운로드 : https://nodejs.org/ (LTS 버전 권장)

Q. 보고서 생성에 너무 오래 걸립니다.
A. LLM API 서버 상태에 따라 2~10분이 소요될 수 있습니다.
   화면을 닫지 말고 기다려 주세요.

Q. "판례를 찾을 수 없습니다" 메시지가 나타납니다.
A. 국가법령정보 API 키가 올바른지 확인하세요.
   Configuration 화면(http://localhost:3000/config)에서 재설정할 수 있습니다.

Q. API 키를 변경하고 싶습니다.
A. http://localhost:3000/config 에서 언제든지 변경 가능합니다.
   저장 후 서버가 자동으로 재시작되어 즉시 반영됩니다.

Q. install.bat 을 더블클릭해도 화면이 바로 닫힙니다.
A. install.vbs 파일을 더블클릭하여 설치하세요.
   일부 PC에서 cmd.exe 실행 오류(0xc0000142)가 발생할 수 있으며,
   install.vbs 는 이를 우회하여 PowerShell을 직접 실행합니다.

Q. 새 버전으로 업데이트하고 싶습니다.
A. install 폴더의 [ update.bat ] 을 더블클릭하세요.
   API 키와 보고서 데이터는 유지된 채로 소스코드만 최신 버전으로 교체됩니다.
   자세한 내용은 7번 "업데이트 방법" 항목을 참고하세요.

Q. 이미 설치한 후 재설치가 필요합니다.
A. install.bat 을 다시 실행하면 됩니다.
   기존 데이터(보고서 이력)는 유지되며, 패키지만 업데이트됩니다.

Q. 데이터는 어디에 저장됩니까?
A. 모든 데이터는 로컬 PC에만 저장됩니다.
   저장 위치 : [설치 폴더]\backend\tax_agent.db (SQLite 데이터베이스)


================================================================================
 10. 문제 해결
================================================================================

■ 포트 충돌 오류 (이미 사용 중인 포트)
  증상 : "Port 8001 is already in use" 또는 "Port 3000 is already in use"
  해결 : stop.bat 실행 후 다시 start.bat 을 실행하세요.
         그래도 해결되지 않으면 PC를 재시작하세요.

■ winget 명령어를 찾을 수 없음
  증상 : "'winget'은(는) 내부 또는 외부 명령, 실행할 수 있는 프로그램..."
  해결 : Windows를 최신 버전으로 업데이트하거나,
         Python과 Node.js를 수동으로 설치 후 install.bat 을 다시 실행하세요.

■ PowerShell 실행 정책 오류
  증상 : "이 시스템에서 스크립트를 실행할 수 없으므로..."
  해결 : install.bat 은 -ExecutionPolicy Bypass 옵션으로 실행되므로
         별도 정책 변경 없이 동작해야 합니다.
         그래도 오류가 발생하면 install.vbs 를 대신 더블클릭하세요.
         install.vbs 도 실패할 경우, 관리자 권한 명령 프롬프트에서
         아래 명령을 입력한 후 install.bat 을 다시 실행하세요.
         > Set-ExecutionPolicy RemoteSigned -Scope CurrentUser

■ install.bat 실행 시 화면이 즉시 닫힘 (0xc0000142 오류)
  증상 : install.bat 더블클릭 시 창이 바로 사라지거나
         "응용 프로그램을 올바르게 시작할 수 없습니다 (0xc0000142)" 오류 발생
  원인 : 일부 환경에서 cmd.exe 자체가 정상 실행되지 않는 경우
  해결 : install.vbs 파일을 더블클릭하세요.
         (cmd.exe를 거치지 않고 PowerShell을 직접 실행합니다)

■ npm run build 실패
  증상 : "프론트엔드 빌드에 실패했습니다"
  해결 : 인터넷 연결을 확인하고 install.bat 을 다시 실행하세요.

■ 백엔드 서버가 즉시 종료됨
  증상 : "세무AI-백엔드" 창이 바로 닫힘
  해결 : backend/.env 파일이 존재하는지 확인하세요.
         없다면 backend/.env.example 을 복사하여 .env 로 저장하세요.

■ 백신 프로그램이 설치를 차단함
  해결 : 백신 프로그램의 실시간 감시를 일시 중지한 후 설치하세요.
         설치 완료 후 다시 활성화하시기 바랍니다.

■ 로그 파일 위치
  백엔드 오류 로그 : "세무AI-백엔드" 명령 프롬프트 창에 출력됩니다.
  오류 내용을 캡처하여 개발자에게 문의하시면 도움을 드릴 수 있습니다.
