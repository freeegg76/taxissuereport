## SCR-001 이슈 입력

### 화면 목적
사용자가 검토할 세무 이슈를 입력하고 분석을 시작하는 진입점.

### 레이아웃

```
┌─────────────────────────────────────────────┐
│  세무 이슈 분석                              │
│  세무 이슈를 자연어로 입력하면 관련 판례를   │
│  검색하고 분석 보고서를 자동 생성합니다.     │
│                                              │
│  제목 (선택)                                 │
│  ┌──────────────────────────────────────┐   │
│  │ 예) 부당행위계산부인 검토             │   │
│  └──────────────────────────────────────┘   │
│                                              │
│  세무 이슈 *                                 │
│  ┌──────────────────────────────────────┐   │
│  │ 예) 법인이 대표이사에게 시가보다 낮은 │   │
│  │ 가격으로 부동산을 양도한 경우...      │   │
│  │                                      │   │
│  └──────────────────────────────────────┘   │
│                                              │
│  [ 분석 시작 ]                               │
└─────────────────────────────────────────────┘
```

### 컴포넌트 목록

| 컴포넌트 | 타입 | 설명 | 유효성 검사 |
|----------|------|------|-------------|
| `IssueInputForm` | Form 컨테이너 | 전체 폼 래퍼 | — |
| `TitleInput` | `<input type="text">` | 이슈 제목 (선택) | 최대 100자 |
| `IssueTextarea` | `<textarea>` | 세무 이슈 본문 (필수) | 최소 10자, 최대 2000자 |
| `CharacterCounter` | 표시 컴포넌트 | 현재 입력 글자 수 / 최대 글자 수 | — |
| `SubmitButton` | `<button>` | "분석 시작", 로딩 중 disabled + 스피너 | IssueTextarea 비어있으면 비활성 |

### 인터랙션 및 API

1. "분석 시작" 클릭 → `POST /api/v1/issues` 호출 (`title`, `raw_input` 전송)
2. 성공 시 `issue_id` 반환 → **SCR-003**으로 자동 이동
3. 실패 시 인라인 에러 메시지 표시

### 상태 관리 (Zustand)

```ts
// issueStore
{
  title: string;
  rawInput: string;
  isSubmitting: boolean;
  error: string | null;
}
```

## SCR-003 이슈 분석 진행 상태

### 화면 목적
이슈 입력 후 LLM 분석 → 판례 검색 → 판례 선별 → 보고서 생성까지 진행 상황을 실시간으로 표시. 사용자가 중간 과정을 인지할 수 있도록 단계별 상태를 노출한다.

### 레이아웃

```
┌─────────────────────────────────────────────┐
│  분석 중...                                  │
│  부당행위계산부인 검토                        │
│                                              │
│  ● STEP 1  이슈 분석           ✅ 완료       │
│    세목: 법인세 / 키워드: 부당행위계산부인...  │
│                                              │
│  ● STEP 2  판례 검색 중        ⏳ 진행 중    │
│    국가법령정보 API 조회 중...                │
│                                              │
│  ○ STEP 3  판례 선별           — 대기        │
│  ○ STEP 4  보고서 생성         — 대기        │
│                                              │
│  [ 취소 ]                                    │
└─────────────────────────────────────────────┘
```

### 컴포넌트 목록

| 컴포넌트 | 타입 | 설명 |
|----------|------|------|
| `ProgressStepper` | 단계 표시 | 4단계 세로 스텝퍼 (완료/진행중/대기/실패) |
| `StepDetail` | 접이식 영역 | 완료된 단계는 요약 정보 노출 (세목, 키워드, 검색된 판례 수 등) |
| `StatusIcon` | 아이콘 | ✅ 완료 / ⏳ 진행중 / ○ 대기 / ❌ 실패 |
| `CancelButton` | `<button>` | 분석 취소, 이슈 입력 화면으로 복귀 |
| `ErrorMessage` | 인라인 메시지 | 단계 실패 시 오류 사유 + "재시도" 버튼 |

### 인터랙션 및 API

| 단계 | API 호출 | 완료 기준 |
|------|----------|-----------|
| STEP 1 이슈 분석 | `POST /api/v1/issues/{id}/analyze` | `tax_category`, `keywords`, `search_strategy` 반환 |
| STEP 2 판례 검색 | `POST /api/v1/issues/{id}/search-cases` | `saved_count > 0` |
| STEP 3 판례 선별 | 백엔드 내부 처리 (LLM) | `SELECTED=1` 판례 1건 이상 |
| STEP 4 보고서 생성 | `POST /api/v1/issues/{id}/report` | `report_id` 반환 |

- 모든 단계 완료 시 **SCR-005**로 자동 이동
- 실패 시 해당 단계에 에러 표시 + "재시도" 버튼

## SCR-006 피드백 입력 및 반영

### 화면 목적
보고서에 대한 수정 요청을 입력하고 즉시 보고서에 반영하는 화면. SCR-005의 우측 슬라이드 패널로 표시.

### 레이아웃 (우측 슬라이드 패널, 너비 400px)

```
┌─────────────────────────────┐
│  피드백 입력            [✕] │
│  ─────────────────────────  │
│  수정 요청 내용 *            │
│  ┌───────────────────────┐  │
│  │ 예) 대법원 판례 위주로 │  │
│  │ 재구성해 주세요.       │  │
│  └───────────────────────┘  │
│                              │
│  [ 반영하기 ]                │
│  ─────────────────────────  │
│  이전 피드백 이력            │
│  · 2026.05.14 15:32         │
│    "결론 섹션 보완 요청"     │
│    → 반영 완료              │
└─────────────────────────────┘
```

### 컴포넌트 목록

| 컴포넌트 | 타입 | 설명 |
|----------|------|------|
| `FeedbackPanel` | 슬라이드 패널 | SCR-005 위에 오버레이, 외부 클릭 시 닫힘 |
| `FeedbackTextarea` | `<textarea>` | 수정 요청 자연어 입력 (필수, 최대 1000자) |
| `SubmitFeedbackButton` | `<button>` | 제출 중 로딩 스피너, 완료 시 패널 닫고 SCR-005 자동 갱신 |
| `FeedbackHistoryList` | 목록 | 이전 피드백 요청 및 처리 결과 표시 (로컬 세션 기준) |
| `SearchingIndicator` | 인라인 상태 | 신규 판례 재검색 중일 때 표시 ("판례 재검색 중...") |

### 인터랙션 및 API

1. "반영하기" 클릭 → `PUT /api/v1/issues/{id}/report/feedback` (`feedback_text` 전송)
2. 응답의 `searched_count > 0` 이면 SearchingIndicator 표시
3. 완료 후 `GET /api/v1/issues/{id}/report` 재호출 → SCR-005 보고서 내용 자동 갱신

### 컴포넌트 목록

### 인터랙션 및 API

2. 성공 시 모달 닫기, SCR-005 상태 배지 `finalized`로 갱신
3. FeedbackButton / FinalizeButton 비활성화

## SCR-008 파일 내보내기 / 다운로드

### 화면 목적
확정된 보고서를 DOCX 또는 PDF로 내보내는 기능. SCR-005 상단 "내보내기" 드롭다운 메뉴로 접근.

### 레이아웃 (인라인 드롭다운 + 진행 토스트)

```
[ 내보내기 ▾ ]
  ├── Word 문서 (.docx)
  └── PDF 문서 (.pdf)

──────────────────────────────
(파일 생성 중 토스트)
  ⏳ DOCX 생성 중...
  ✅ 다운로드 준비 완료  [ 다운로드 ]
```

### 컴포넌트 목록

| 컴포넌트 | 타입 | 설명 |
|----------|------|------|
| `ExportDropdown` | 드롭다운 메뉴 | DOCX / PDF 선택 |
| `ExportToast` | 토스트 알림 | 생성 진행 중 / 완료 / 실패 상태 표시 |
| `DownloadButton` | `<button>` | 생성 완료 후 활성화, `GET /api/v1/files/{file_id}/download` 호출하여 파일 다운로드 |

### 인터랙션 및 API

| 선택 | API 호출 | 완료 후 |
|------|----------|---------|
| Word (.docx) | `POST /api/v1/issues/{id}/report/export/docx` | `download_url` 반환 → 자동 다운로드 |
| PDF (.pdf) | `POST /api/v1/issues/{id}/report/export/pdf` | `download_url` 반환 → 자동 다운로드 |

## SCR-009 Configuration

### 화면 목적
LLM Provider, LLM 모델, 국가법령정보 API 키를 설정하는 화면. 사이드바 하단 "Configuration" 링크로 진입.

### 레이아웃

```
┌─────────────────────────────────────────────┐
│  Configuration                               │
│                                              │
│  LLM Provider                                │
│  ┌──────────────────┐                       │
│  │ Anthropic     ▼  │  (Anthropic / Gemini) │
│  └──────────────────┘                       │
│                                              │
│  LLM 모델                                    │
│  ┌──────────────────┐                       │
│  │ claude-sonnet-4  │                       │
│  └──────────────────┘                       │
│                                              │
│  LAW_API_KEY                                 │
│  ┌──────────────────────────────────────┐   │
│  │ ••••••••••••••••••••          [ 확인 ]│   │
│  └──────────────────────────────────────┘   │
│                                              │
│  [ 저장 ]                        OK         │
└─────────────────────────────────────────────┘
```

### 컴포넌트 목록

| 컴포넌트 | 타입 | 설명 |
|----------|------|------|
| `ConfigForm` | Form 컨테이너 | 설정 전체 래퍼 |
| `LLMProviderSelect` | `<select>` | Anthropic / Gemini 선택 |
| `LLMModelInput` | `<input>` | 선택한 Provider 기준 모델명 입력 |
| `ApiKeyInput` | `<input type="password">` | LAW_API_KEY 입력, 마스킹 처리 |
| `ApiKeyVerifyButton` | `<button>` | API 키 유효성 확인 (선택) |
| `SaveButton` | `<button>` | 설정 로컬 저장 (localStorage 또는 서버 저장) |
| `SaveSuccessIndicator` | 인라인 텍스트 | 저장 완료 시 "OK" 표시 |

### 저장 방식
- 설정값은 브라우저 `localStorage`에 저장하거나 별도 설정 API 엔드포인트로 전달 (구현 시 결정)
- API 키는 화면에 마스킹 표시, 평문 노출 최소화
